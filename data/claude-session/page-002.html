Yes, confirmed — that's exactly the design, but it only works because of two specific structural choices I want to call out so you can sanity-check them.

Why the isolation holds

Docker BuildKit (default in modern Docker) caches each stage independently and only invalidates layers from the point an input changes onward. So:

  • The frontend-builder stage never COPYs anything from the backend → backend .py edits cannot reach it, so it stays fully cached.
  • The runtime stage does pull from the frontend builder, BUT only as the last COPY step. So frontend changes invalidate that final copy and nothing earlier.

Layer order (the part that makes it work)

# Stage 1: frontend-builder (node:22-alpine)
COPY frontend/package.json frontend/yarn.lock ./
RUN corepack enable && yarn install --immutable   # cached unless deps change
COPY frontend/ ./                                  # invalidated by FE src edits
RUN yarn build

# Stage 2: runtime (python:3.14-slim)
COPY backend/pyproject.toml backend/uv.lock ./
RUN uv sync --frozen --no-dev                      # cached unless BE deps change
COPY backend/ ./                                   # invalidated by BE src edits
COPY --from=frontend-builder /app/dist ./static    # invalidated by FE changes
EXPOSE 8000
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]

What invalidates what

Change yarn install yarn build uv sync runtime COPY backend/ runtime COPY --from=
BE .py edit cached cached cached rebuild cached
FE .vue/.ts edit cached rebuild cached cached rebuild
package.json rebuild rebuild cached cached rebuild
pyproject.toml cached cached rebuild rebuild rebuild

Two subtleties worth knowing:

  1. RUN uv sync must come BEFORE COPY --from=frontend-builder in the runtime stage. If I swapped them, FE edits would bust the Python deps cache. My plan keeps the order above.
  2. With BuildKit, the frontend-builder and runtime-stage's Python-deps install can run in parallel — they're independent until the final COPY --from=. So a cold build is also faster.

  execFileSync('docker', ['rm', '-f', name], { stdio: 'ignore' })

  // No such container — nothing to clean up.
